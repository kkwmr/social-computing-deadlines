## Data

Machine-readable snapshots live in [`data/`](data/):

| File | Format | Contents |
|---|---|---|
| [`deadlines.json`](data/deadlines.json) | JSON | One object per venue-year: full name, location, conference dates, official website, per-deadline entries (`label`, ISO-8601 `date`, `tz`), status, and source URLs |
| [`deadlines.csv`](data/deadlines.csv) | CSV | One row per deadline: `venue, year, area, deadline_label, deadline_date, timezone, conference_dates, place, website, status, source` |

The same data powers the live site at [casp.jp/social-computing-deadlines](https://casp.jp/social-computing-deadlines/).

## Contributing

Corrections and venue suggestions are welcome — please [open an issue](../../issues) or a pull request against the files in `data/`. Two ground rules:

1. Every date must cite an **official source URL** (CfP page, organizer site, or organizer announcement).
2. Estimated dates (not yet officially announced) must be marked `status: estimated`.

Accepted changes are folded into the canonical dataset and appear on the live site and in this mirror with the next daily sync.
